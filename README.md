# Night Adventure

**Night Adventure** — квест-приключенческая игра с изометрической графикой, разработанная на платформе «Урсула» в рамках производственной практики. Игрок помогает привидению найти потерянный дом, проходя через испытания, сражаясь с монстрами и принимая решения, влияющие на сюжет.

---

## 🎮 Сюжет и механики

Игра начинается с того, что главный герой встречает привидение, которое просит помочь отыскать дорогу к своему домику в лесу. Игроку предстоит:
- Выбирать маршрут (правильный путь отмечен оранжевыми указателями).
- Взаимодействовать с объектами (подобрать фонарик, оружие, ключ).
- Сражаться с врагами (злые мухи, монстр).
- Общаться с NPC (сторож кладбища, волшебник) через диалоговую систему.

В игре реализовано несколько альтернативных концовок, зависящих от действий игрока.

---

## 🛠 Используемые технологии

- **Платформа «Урсула»** — создание изометрических сцен, визуализация, физика.
- **ПРИМС** — визуальный язык программирования (иерархические машины состояний) для логики поведения объектов, NPC и врагов.
- **Ink** — язык сценариев для диалогов и сюжетных веток.
- **Blender 2.83** — редактирование и адаптация 3D-моделей (формат GLB).

## 🚀 Как запустить игру

Для запуска игры требуется доступ к платформе «Урсула». Выполните следующие шаги:

### 1. Подготовка файлов
Скачайте содержимое репозитория (все файлы и папки) на свой компьютер.

### 2. Загрузка в редактор игр
1. Перейдите на платформу «Урсула» ([games.kruzhok.org](https://games.kruzhok.org/)).
2. В личном кабинете выберите **«Создать игру»** или откройте существующий проект.
3. В редакторе игр перейдите в раздел управления ресурсами (обычно это вкладка **«Модели»**, **«Сценарии»** или **«Графы»**).

### 3. Импорт компонентов
Последовательно загрузите файлы из репозитория:

> **Примечание:** Сами 3D-модели (GLB-файлы) в репозиторий не включены из-за ограничений по размеру. Вы можете использовать собственные модели или найти подходящие на внешних ресурсах (например, Sketchfab).

### 4. Настройка сцены
1. В редакторе сцен расставьте объекты (герой, NPC, враги, интерактивные предметы) на игровом поле.
2. Для каждого объекта назначьте соответствующий граф поведения (ПРИМС) из папки `/graph`.
3. Привяжите сценарий (Ink) к NPC, которые ведут диалоги.
4. Настройте коллизии и триггеры для проходимых/непроходимых зон.

### 5. Запуск и тестирование
Нажмите кнопку **«Запустить»** или **«Play»** в редакторе, чтобы проверить игру. Убедитесь, что:
- Все диалоги открываются корректно.
- Переходы между сценами и ветки сюжета работают.
- Объекты не проваливаются сквозь пол (при необходимости скорректируйте коллайдеры в настройках модели).
